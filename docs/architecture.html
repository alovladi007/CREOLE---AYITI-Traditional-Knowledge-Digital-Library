üõ†Ô∏è Technology Stack

Frontend

  • Framework: Next.js 14
  • UI Library: React 18
  • Language: TypeScript
  • Styling: Tailwind CSS
  • State: React Query
  • i18n: next-i18next
  • Theme: next-themes
  • Maps: Leaflet
  • Charts: Chart.js
  • Editor: Quill.js

Backend

  • Framework: NestJS
  • Language: TypeScript
  • ORM: TypeORM
  • API: REST + GraphQL
  • Validation: class-validator
  • Documentation: Swagger
  • Rate Limiting: ThrottlerModule
  • Jobs: Bull Queue
  • Cache: cache-manager

Database

  • RDBMS: PostgreSQL 15
  • Full-Text Search: tsvector/tsquery
  • JSONB: For flexible data
  • Indexes: B-tree, GIN, GiST
  • Migrations: TypeORM migrations
  • Pooling: Connection pool

Authentication

  • Provider: Keycloak
  • Protocol: OAuth2/OIDC
  • Flow: Authorization Code + PKCE
  • 2FA: TOTP (RFC 6238)
  • Tokens: JWT
  • SSO: Supported

Caching & Jobs

  • Cache: Redis 7
  • Queue: Bull (Redis-based)
  • TTL Strategies: 5m, 1h, 1d
  • Cache Manager: NestJS cache-manager
  • Background Jobs: Async processing

Storage & Media

  • Object Storage: MinIO
  • Protocol: S3-compatible
  • Processing: OCR, Transcription
  • Image: Sharp, EXIF extraction
  • Video: FFmpeg transcoding

DevOps

  • Containerization: Docker
  • Orchestration: Docker Compose
  • CI/CD: GitHub Actions ready
  • Monitoring: Health checks
  • Logging: Winston, Pino

NLP Service

  • Language: Python 3.11
  • Framework: FastAPI
  • NLP: spaCy, NLTK
  • Creolist: Haitian Creole processing
  • Models: Custom trained models

Scalability Considerations

  • Horizontal Scaling: Frontend and backend can run multiple instances behind load balancer
  • Database: PostgreSQL read replicas for read-heavy workloads
  • Cache: Redis cluster for high availability
  • Storage: MinIO distributed mode for multi-node setup
  • CDN: Static assets can be served via CDN
  • Async Jobs: Bull workers can be scaled independently
